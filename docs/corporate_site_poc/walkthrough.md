# Header リンク動的生成 完了報告 (Walkthrough)

ご要望を受け、`src/components/Header.astro` のナビゲーションリンクを `src/pages` 内のファイルから自動で生成する仕組みに変更しました。

## 実装のポイント (Changes Made)

単なる全取得ではなく、以下のようなプロセスを踏んでいます。

### 1. ファイル一覧の取得
Vite の機能である `import.meta.glob('/src/pages/**/*.astro')` を用い、ディレクトリ内の対象ファイルを全て拾い上げます。

### 2. 動的ページの除外フィルタリング
取得したファイルパスの中で、`news/[id].astro` や `news/[...page].astro` のように `[` (ブラケット) が含まれる動的ルーティング用のファイルは、リンクとしても成立しないため `filter` 処理で除外するようにしました。

### 3. パスの整形とラベルの生成
ディレクトリのパス (`/src/pages/contact.astro`) を実際のURL (`/contact`) に整形し、同時にファイル名から頭文字を大文字にしたラベル (`Contact`) を自動で生成する仕組みを追加しました。
※ `index.astro` は特別扱いとし、URLを `/`、ラベルを `Home` としています。

### 4. アンカーリンクとのマージ (ハイブリッド方式)
ページ内の特定セクションへ飛ぶためのアンカーリンク (`/#services` や `/#about`) はファイルとしては存在しないため、自動取得リストとは別に「固定リンク (Static Links)」として定義しました。
これにより、**「自動取得したページ」＋「固定のページ内リンク」** を結合して一つのメニューとして表示するようにしています。

## 検証結果 (Verification)
- `npm run build` を実行し、動的生成処理でエラーが発生しないこと、ビルドが正常に完了することを確認済みです。
- UI上もこれまで通り "Home", "Services", "About", "News", "Contact" のナビゲーションが指定の順序で正しく展開されるようになっています。
