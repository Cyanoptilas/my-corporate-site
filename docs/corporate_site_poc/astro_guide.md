# 初心者向け Astro 入門・技術解説ガイド

このドキュメントでは、本プロジェクトで使用している「Astro」について、初心者が理解できるよう、基本概念から実際のコードの使われ方まで詳しく解説します。

---

## 1. Astro とは何か？

Astro (アストロ) は、**「コンテンツ駆動型」**のWebサイト（コーポレートサイト、ブログ、ポートフォリオサイトなど）を超高速に表示するためのモダンなWebフレームワークです。

### 1.1 最大の特徴: Zero JS アーキテクチャ
React, Vue, Svelte などの一般的なフレームワークは、ページを表示するために裏側で大量の JavaScript (JS) を読み込みます。しかし、大部分のサイト（特に記事や会社情報）において、初期表示に JavaScript は不要です。
Astro は、ビルド時に HTML を生成し、**不要な JavaScript をデフォルトで全て取り除く (Zero JS)** ため、初期読み込みが爆速になります。

### 1.2 アイランドアーキテクチャ (Astro Islands)
「じゃあ、お問い合わせフォームみたいにJSが必要なときはどうするの？」
Astro は「アイランドアーキテクチャ」という仕組みを持っています。ページ全体は静的な HTML で構成しつつ、インタラクティブ（動的）にしたい部分だけを「島（Island）」として扱い、そこにだけ JavaScript を読み込ませることができます。

## 2. Astroの基本： `.astro` ファイルの構造

本プロジェクトの `src/components/Header.astro` や `src/pages/index.astro` などのファイルは、Astro独自のコンポーネントファイルです。
3つのセクションに分かれています。

### ① コンポーネントスクリプト (Component Script)
ファイルの冒頭の `---`（コードフェンス）で囲まれた部分です。
ここでデータ取得（API通信）や、変数の定義、他コンポーネントのインポートを行います。ここは**サーバー側でのみ実行**され、ブラウザにはこのコードは送られません。

```astro
---
// ここがコンポーネントスクリプト
import Layout from "../layouts/Layout.astro";
import { getNewsList } from "../lib/microcms";

const { contents: news } = await getNewsList({ limit: 3 });
---
```

> [!IMPORTANT]
> **データ取得のタイミング（SSGによる爆速表示の理由）**
> この `await getNewsList(...)` によるデータ取得は、**ビルド時（`npm run build` を実行した時）に1回だけ**行われます。
> そのため、ユーザーがサイトにアクセスした瞬間には「すでにデータが埋め込まれた完成済みのHTML」が存在しており、API通信の待ち時間ゼロで表示されるという大きなメリットがあります（これを SSG: 静的サイト生成 と呼びます）。
> 
> ※ **逆に言うと**: ビルド後に microCMS の管理画面でニュースを追加しても、すぐにはサイトに反映されません。
> 新しい記事を追加した際に自動でサイトを更新するには、microCMS 側に **Webhook（更新されたことを Cloudflare に通知し、再ビルドを走らせる仕組み）** を設定するのが一般的です。
> 
> ※ **AstroのSSR/Hybridレンダリングについて**:
> `astro.config.mjs` の設定で `output: 'server'`（SSRモード）にすると、デフォルトで**すべてのページ**がアクセスされるたびにリクエストを受け取って動的に生成されるようになります。
> ただし、`output: 'hybrid'` に設定すると、「基本はSSG（ビルド時生成）だけど、特定のページ（例: 検索ページやログインページ）だけSSRにする」という**いいとこ取り**の設定が可能です。プロジェクトの性質に合わせて使い分けます。

### ② コンポーネントテンプレート (HTML + JSX風)
`---` の下の部分です。HTMLを基盤としつつ、波括弧 `{ }` を使って JavaScript の変数を埋め込んだり、配列をループ（`.map`）させたりできます。

```astro
<div class="news-list">
  {news.length === 0 ? (
    <p>お知らせはありません。</p>
  ) : (
    news.map((item) => <h3>{item.title}</h3>)
  )}
</div>
```

### ③ スタイルタグ (Style)
`<style>` タグを書くと、自動的に**そのコンポーネントにだけ適用される「スコープ付きCSS」**になります。（※今回のプロジェクトでは主に Tailwind CSS を使用しているため、`<style>` タグはあまり使われていません）。

## 3. 本プロジェクトでの Astro の具体的な使われ方

### 3.1 共通レイアウト (`src/layouts/Layout.astro`) と `<slot />`
複数のページで共通のヘッダーやフッター、`<head>` タグ（メタ情報など）を使い回すための仕組みです。
- `<slot />` という魔法のタグの部分に、このレイアウトを使用したページ（例: `index.astro`）の中身がはめ込まれます。

```astro
<!-- Layout.astro の抜粋 -->
<body>
  <Header />
  <main>
    <slot /> <!-- 個別のページの内容がここに入る -->
  </main>
  <Footer />
</body>
```

### 3.2 高速な画面遷移：View Transitions (`<ClientRouter />`)
通常、リンクをクリックすると画面全体が白くなってから次のページが表示されます。
Astro では `Layout.astro` の `<head>` に `import { ClientRouter } from "astro:transitions";` を読み込み `<ClientRouter />` を配置するだけで、**SPA（Single Page Application）のような、ぬるっとした画面遷移**を自動で実現してくれます。

### 3.3 動的ルーティング (`getStaticPaths`)
ブログやコーポレートサイトの「お知らせ詳細」は、記事の数だけURLが存在します。（例: `/news/1`, `/news/2`）
これらを自動生成する仕組みが `src/pages/news/[id].astro` です。

ファイル名にブラケット `[ ]` を使うと「動的なURL」になり、中の `getStaticPaths` 関数で「どのURLのページを生成するか（ここでは microCMS から取得した記事ID一覧）」をAstroに教え、ビルド時に全ページを作り出します。

### 3.4 ページネーション (`paginate`)
`src/pages/news/[...page].astro` ではお知らせ一覧を「1ページあたり10件」等で分割しています。
`Astro` に標準搭載されている `paginate()` 関数を使うことで、「次へ」「前へ」といったページング処理を非常に簡単に実装できます。ファイル名に `[...page]` を使うのもAstroのルールです。

## まとめ

Astro を使うことで、このプロジェクトは：
- バックエンド（microCMS）からのデータ取得を簡単に行える。
- 余計なJSを省くため、非常に高速に表示される。
- コンポーネント指向でUIを整理できる。
- URLごとにページを出し分けるルーティングがファイルシステム（フォルダ構成）ベースで直感的に作れる。

という恩恵を受けています。まずは `.astro` ファイルの `---` の中身（データ準備）と下半分（HTMLのレンダリング）の関係性を理解することが第一歩です！
