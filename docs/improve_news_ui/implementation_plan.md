# ニュースページUI改善計画

ニュース一覧ページ (`/news`) の閲覧性を向上させるため、デザインを大幅に刷新します。

## 課題と方針
現状のニュース一覧はシンプルなリスト形式ですが、視覚的な魅力に欠けています。これを**画像（アイキャッチ）付きのカード型グリッドレイアウト**に変更し、よりモダンで閲覧しやすいデザインにします。

## Proposed Changes

### フロントエンド / UIコンポーネント

#### [MODIFY] [...page].astro (file:///c:/Development/my-corporate-site/src/pages/news/[...page].astro)
- **データの取得追加**: `getStaticPaths` 内の `fields` 指定に `eyecatch` を追加し、一覧画面でもアイキャッチ画像を取得できるようにします。
- **レイアウトの変更**: リスト表示からレスポンシブなCSSグリッド (`grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8`) に変更します。
- **カードデザインの実装**:
  - 各記事をカードコンポーネントとしてスタイリングします。
  - アイキャッチ画像が存在する場合は上部に表示し、無い場合はグレーのプレースホルダー（またはロゴ等の代替表示）を配置します。
  - ホバー時にカード全体が少し浮き上がるようなマイクロインタラクション（`transition-transform hover:-translate-y-1 hover:shadow-lg` など）を追加し、クリックできることを直感的に伝えます。
  - 日付やカテゴリバッジの配置を整理し、タイトルとのバランスを整えます。

## Verification Plan

### Automated Tests
- 本プロジェクトには現在フロントエンドの自動テストフレームワークが設定されていないため、手動での確認を主とします。
- 開発サーバーのビルドエラーが出ないことを確認します。

### Manual Verification
1. `npm run dev` でローカルサーバーを起動します。
2. ブラウザで `http://localhost:4321/news` にアクセスします。
3. ニュース一覧が3カラム（PCの場合）のカード型で表示されていることを確認します。
4. 画像が正しく表示され、画像がない場合でもレイアウトが崩れないことを確認します。
5. スマホサイズにブラウザ幅を縮めた際、1カラムのレイアウトに適切にレスポンシブ変化することを確認します。
6. カードをクリックし、ニュース詳細ページ (`/news/[id]`) へ正しく遷移することを確認します。
